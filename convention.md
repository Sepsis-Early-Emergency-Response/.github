# Git Convention

본 문서는 프로젝트 전반에서 사용하는 **Commit Message**, **Branch**, **Pull Request** 규칙을 정의한다.  
모든 구성원은 아래 규칙을 따른다.

---

## Commit Message

```text
label: message
```

### Rules

1. 카테고리 별로 나누어진 **라벨(label)** 을 앞에, 커밋 내용을 담은 **메시지(message)** 를 뒤에 둔다.
2. 모든 커밋 메시지는 **소문자**로 작성한다.
3. 필요할 경우 **kebab-case**를 따른다.
4. 커밋 메시지는 **영문으로 통일**한다.
5. 커밋 메시지는 가능한 한 **어디서 / 무엇을 / 어떻게 했는지** 알아볼 수 있도록 작성한다.

---

### Description

1. 커밋 설명글은 **필수 요소가 아니다**.
2. 필요하지 않을 경우 **공란으로 둔다**.
3. 작성이 필요한 경우 **영문으로 구체적으로 작성**한다.

---

## Branch

```text
label/domain
```

### Rules

1. 앞에 카테고리별 라벨을 붙이고, 뒤에 작업한 도메인을 작성한다.
2. 브랜치 간 병합을 수행할 경우 반드시 **코드 리뷰어의 최종 확인 및 approve**를 거친다.
3. 병합은 **브랜치 생성자**가 수행한다.
4. 병합 시 커밋 메시지는 다음 규칙을 따른다.
    - 일반 브랜치 병합: `merge: branch name`
    - develop 브랜치를 main에 병합하는 경우: `merge: release [version]`

## Commit Label

| Label  | Description |
|--------|-------------|
| feat   | 추가 기능 개발 / 새로운 코드 추가 |
| fix    | 기능/코드 수정 |
| bug    | 버그/에러 수정 |
| merge  | 브랜치 병합 |
| deploy | 프로젝트 배포 / 관련 문서 작업 |
| docs   | 문서 추가 / 수정 |
| delete | 코드 / 파일 / 문서 삭제 |
| note   | 주석 추가 / 제거 |
| style  | 코드 스타일 / 구조 수정 |
| config | 기초 설정 파일 / 의존성 / 라이브러리 설정 수정 |
| etc    | 기타 위에 해당하지 않는 경우 |
| tada   | 프로젝트 생성 |
